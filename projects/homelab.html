Homelab: Build Guide (Oracle VM)

One-line summary: Reproducible homelab setup using Oracle VM VirtualBox to run isolated test networks for pentesting, detection testing, and training.

Role, timeframe, tools

  • Role: Lab architect / systems engineer
  • Timeframe: 2 weeks (initial build)
  • Tools: Oracle VM VirtualBox, Vagrant (optional), Linux (Ubuntu 22.04), Windows 10, pfSense, WireGuard, Docker, Ansible (automation snippets)

Objective

Deliver a reproducible, segmented homelab for offensive and defensive testing: attacker VLAN, target VLAN, management host, and simulated internet (NAT). Provide hardening steps and automation snippets so others can reproduce safely.

Topology (high level)

Simple topology:

Attacker (Kali) -- Host-only net 192.168.56.0/24
Target VMs (Metasploitable, Win7) -- Host-only net 192.168.56.0/24
Management (host) -- NAT to internet
Optional pfSense VM controls routing between NAT/host-only networks

If you prefer an image, add assets/homelab-topology.png and link it here.

Step-by-step (essential)

  1. Create host-only network in VirtualBox (example): 
    VBoxManage hostonlyif create
VBoxManage hostonlyif ipconfig vboxnet0 --ip 192.168.56.1
  2. Import VM images / ISO:
    • Ubuntu Server 22.04 (as management/utility)
    • Kali Linux (attacker)
    • Metasploitable (target)
    • pfSense (router, optional)
  3. Configure VM NICs: 
     # Example VirtualBox NIC setup (attach 1: NAT, 2: Host-only)
VBoxManage modifyvm "Kali" --nic1 nat --nic2 hostonly --hostonlyadapter2 vboxnet0
  4. Basic hardening checklist:
    • Disable unnecessary services on management VMs
    • Snapshot golden images: `VBoxManage snapshot take "clean_install"`
    • Use host-only networks for lab traffic
    • Restrict host firewall rules so lab traffic cannot escape unintended networks
  5. Automation example (Ansible playbook snippet that updates packages): 
    - hosts: homelab
  become: true
  tasks:
    - name: Update apt cache
      apt:
        update_cache: yes
    - name: Upgrade packages
      apt:
        upgrade: dist

Findings and recommended baseline

Key takeaways for a usable homelab:

  • Keep attack and target networks logically separated with host-only adapters; expose only required services
  • Use snapshots before risky exercises to revert quickly
  • Document IP addressing and services in each VM; include a README per VM
